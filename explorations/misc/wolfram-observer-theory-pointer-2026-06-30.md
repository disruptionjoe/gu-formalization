## Why this is only a pointer, not an obligation

Wolfram's program is a *computational* foundation (hypergraph rewriting, the ruliad, observers as
equivalencing/coarse-graining, the second law from observer boundedness). GU is a *geometric-unification*
program (tetrad/observerse, index theory, generation structure). The two are largely orthogonal: unlike the
quadratic-gravity reading set (Stelle / Avramidi-Barvinsky / Turok), Wolfram's claims do not land on GU's
specific obligations (renormalizability, ghost/unitarity, generation count, no-go index structure). There is
no derivation here that GU must answer.

## The two genuine (weak) contact points

1. **Computational irreducibility / causal structure.** Wolfram's computational-irreducibility and
   causal-invariance themes are adjacent to the existing GU explorations `ca-computational-irreducibility/`
   and `sorkin-causal-set/`. If those lanes ever need an external "computation-first" foil, Wolfram is the
   canonical source. No action required now.
2. **Meta-posture pressure.** Wolfram's "the ruliad runs all rules; we perceive a slice set by our nature"
   is a stance under which *any* specific unification (GU included) is a perceived slice rather than the
   fundamental rule. This is a philosophical foil for GU's posture, not a technical obligation. RESEARCH-POSTURE
   already handles "GU as generative test case"; nothing to change.
