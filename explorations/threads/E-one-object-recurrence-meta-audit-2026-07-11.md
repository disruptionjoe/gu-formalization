## (2) The two readings, argued adversarially

### HEALTHY reading (genuine single degree of freedom)
1. **The coarse object is stable.** Since 2026-06-26 (RS_GU^phys) through 2026-07-11
   (SG4 / the source action), the bottleneck has carried the SAME name: the unbuilt
   RS/IG source action. It has not been swapped for an unrelated object. 17/18
   occurrences resolve to data of that one action.
2. **The convergence is a TEST, not rhetoric.** `anchored-leads-screen-RESULTS.md` ran
   six independent sweep leads (tmf-elliptic, km-n>=3, Spin(8)-triality, bordism inflow,
   ...) and asked of each: does it escape the gate or gate on the SAME source action?
   All six gated on it; none escaped. A program that were merely re-labelling its
   ignorance would not survive an adversarial "does any route escape?" screen with a
   clean zero. That is the signature of a real single obstruction.
3. **One construction discharges all sub-data at once.** c_W, the SG4 2-bit, and the
   v_PSB sub-block are different *entries* of one action; writing that action fixes them
   simultaneously. "One object decides everything remaining" is then literally true at
   the level of the construction problem, even if the object has internal structure.

### DEGENERATING reading (Lakatos displacement)
1. **"One object" is a conjunction, not an identity -- the repo says so itself.**
   `one-residual-complete-picture` note #6: *"The residuals are jointly fixed by a
   single object ... not literally 'the same object'."* c_W is a **coefficient**; the
   SG4 2-bit is a **field-space declaration**; v_PSB is a **vacuum selection**. These
   are three different data. The unifying claim reduces to "they all live in the same
   action" -- which note #7 concedes is near-truism ("a theory is its action").
2. **The terminus relocates at ever-finer granularity while nothing gets built.**
   Trace the gravity leg on 2026-07-11 alone: whole Y14 ambient geometry -> "one ambient
   contraction (R^Y.B)" -> R^Y computed -> "one structural coefficient" -> c_W -> "the
   OQ2-A functional binary |H|^2 vs |II|^2" -> "welded to the theta-sector." Five
   re-descriptions in one day, each announced as the new "one remaining X," none of them
   the object actually being written. That is the classic degenerating-program move: the
   hard part is displaced one step and re-badged as progress.
3. **The M^2/r^2 episode is the tell.** A flagged residual was first "a genuine canon
   tension," then same-day "resolved to M^2/r^6 -- a convention artifact." A healthy
   single-DOF program does not usually generate and dissolve a crisis inside one session;
   a narrative optimizing for "it fits" does. (The repo caught it -- see below -- which
   counts in its favor, but the episode is evidence the terminus is soft.)
4. **The one hard displacement is real.** 1/18 occurrences (the Y14 ambient contraction)
   named a genuinely NEW geometric object as "the remaining input." It was then computed
   and folded back in -- but that is exactly what displacement looks like when the
   program is productive enough to keep computing the next object.

## (3) Honest verdict -- a SPLIT, graded by level

**At the COARSE level the pattern is HEALTHY.** The bottleneck is a genuine single
unbuilt object -- the RS/IG source action -- and this is backed by a real adversarial
convergence test (anchored-leads: 6 routes, 0 escapes), not by assertion. The object
has been stably named for two weeks and is precisely characterized (SG4 forcing rubric:
a rigid, finite, ~2-bit declaration). This is not a program relabelling a vacuum; it is
a program that has correctly localized its one hole.

**At the FINE level the recent (2026-07-11) narrowing arc shows the DEGENERATING
SIGNATURE.** The "one remaining scalar/convention" language on the gravity leg displaced
through coefficient -> functional -> "welded to theta" multiple times in a single day
while the object stayed unbuilt, and the unifying "one object" is a **conjunction of
distinct data** (coefficient c_W != declaration SG4 2-bit), which the program's own
draft-of-record concedes. The fine-grained "reduced to one c_W" claims are therefore
softer than they read: they are reductions *within* an unbuilt object, and each new pass
manufactures a fresh "one remaining X."

**Why this is not fatal, stated fairly:** the program is *self-aware* about exactly this
failure mode. NEXT-STEPS:67 logs this thread (E1) as a standing watch-item in the same
Lakatos language; note #6 flags the conjunction; the M^2/r^2 crisis was caught and
downgraded, not buried; the Willmore arc is graded "order-grade structural map, NOT a
closed selection theorem." A degenerating program that *announces its own degeneration
criterion and grades itself down* is behaving honestly. The danger is not present
dishonesty; it is the **standing temptation** to keep emitting "reduced to one X" passes
that feel like progress while the one construction that would actually decide anything
(write S_IG) is never attempted, because a free build p-hacks and no forced build is
known.

not free) and check whether it fixes c_W AND the SG4 2-bit AND v_PSB simultaneously with

displacement, confirmed. Until that construction is attempted, every further "reduced to
one X" pass on an already-reduced leg should be treated as **zero new evidence** for the
single-DOF thesis.
