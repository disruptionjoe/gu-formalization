The CRT prime-split axis and the Ehresmannian/Riemannian axis are not the same axis; treating them as one is where the pretty story overreaches the math.

## 3. The adapter mechanism

The one genuinely load-bearing physical fact behind the whole reading is index-type, not arena membership. A Rarita-Schwinger field `Psi_mu` carries a **tangent vector index** `mu` (Riemannian-side, soldered to the manifold frame) tensored with a **spinor index that is gauge-twistable** (Ehresmannian-side). The campaign confirms this is structurally unique: of all campaign objects, only RS reaches the non-frame-trivial row (c) with frame charge `33.94`, "genuinely not of the form `id_14 (x) U`," while the pure-tangent carrier `Lambda^2_+` (vectorlike, no chiral leg) and the pure-fiber chiralizer `C = J.G` (frame charge `0.00`, no tangent leg) each sit entirely on one side (forcing-slot-toy-rs-RESULTS.md). **That uniqueness is the restatement of computed fact** -- and it is the honest reason the count was always gated on the RS source action: it is the only field touching both geometries at once, so it is the only place a count could have crossed. **The interpretation** is reading the forcing-slot null result -- every computed integer is 2-primary (`256 = 2^8`, `-672 = -2^5.3.7`) or the `Z/3` identity (`-42 == 0 mod 3`), and `16(-42) + 3 ch2(V) == 0 mod 3` for every twist -- as "the adapter exists but routes the count signal to the wrong arena." This phrasing is faithful to the 2-primary fact but it silently identifies the 2-primary `Z/8` summand with the Ehresmannian floor, which is false: the tangential/Riemannian-side carrier has its *count* in the 3-primary arena and its *selector* in the 2-primary arena, so the two arenas already mix gauge-side and metric-side objects. Worse, "routes" oversells: RS fails row (a) outright (net self-dual minus anti-self-dual `= 0`, so it carries no net `p_1` and never reaches the `-p_1/24` channel where the order-3 lives), and the only nonzero twisted integer `256 = 2^8` is a tautological projector-sandwich trace, not a measured crossing. So the precise honest statement is: **RS is the unique object that touches both geometries (computed); it is the gated object for that reason (computed + canon); and where its index opens at all, it opens onto the 2-primary selector arena (computed). Calling that "the adapter routing the count to the wrong geometry" is interpretation that fuses two orthogonal axes.**

## 4. The disjointness leg: honestly interpretive-collapsing-to-metaphor, not load-bearing

CRT-disjointness (`Hom(Z/8, Z/3) = Hom(Z/3, Z/8) = 0`) being "the same structural fact" as the disallowed Einstein contraction is **not load-bearing, and on the literal wording it is metaphor.** The reason is the homotopy-vs-geometry category question stated plainly: coprimality is an arithmetic property of the integer 24 that holds "before any physics" and would hold in a universe with any geometry whatsoever; the contraction obstruction is a representation-theoretic naturality fact (no canonical `ad(P) -> Lambda^2 TM` bundle map without soldering) that holds for any structure group at any orders. Neither implies the other. They are aligned only by the contingent, reconstruction-grade frame-charge assignment that happens to drop the gauge object in the 2-primary summand and the tangent object in the 3-primary summand. The leg also names the **wrong lock**: the genuine homotopy analog of "you can't contract the internal index against the tangent index" is the *prime-independent* frame-charge/soldering obstruction (`id_14 (x) U` traceless on the tangent frame = exactly the disallowed move), which would block selector -> carrier even if the orders shared a prime. CRT-disjointness is a redundant arithmetic second lock -- the shadow of coprimality, not of soldering. **Separating test:** find a stable stem `pi_n^s` where an obstruction class and a candidate carrier share a prime block (so `Hom != 0`, coprimality OFF), then compute the internal-fiber selector's frame charge. If frame-charge-0 *still* blocks selector -> carrier, the soldering obstruction is the real, prime-independent analog (and CRT-disjointness was decorative); if sharing a prime *unblocks* the crossing, the whole degree-3 separation was only ever arithmetic and the contraction identification is metaphor confirmed.

## 5. The torsion direction: a grounded probe with a predicted refute -- direction, not result

"Look for the count in the torsion sector" rests on a real and attractive premise -- GU's distortion `theta = A - g.Gamma_LC` *is* torsion in Weinstein's own contortion sense, torsion enters the *source* action, and the gravitino sources torsion in supergravity, so the torsion sector and the (gating) RS source action may be the same uncomputed object. **But as a prediction about where the generation count lives it is false-by-equivocation**, and the value of running it is to close the pun computationally rather than to find the count. **What would be built:** (i) the Dirac / Rarita-Schwinger index coupled to the GU distortion connection `theta` with torsion turned ON, compared against the already-computed torsion-free value; (ii) the Nieh-Yan number `int_M (T^T - e ^ e ^ R)` with its 3-part extracted. **Confirm** (would upgrade to a real route): the torsionful index differs from the torsion-free index by a 3-primary amount -- the gravitino index moves off `-42 == 0 mod 3` when `theta` switches on -- or Nieh-Yan carries odd-torsion content. **Refute** (predicted, and essentially forced by theorem): the index is unchanged mod 3, because Atiyah-Singer is invariant under affine/continuous deformation of the connection, so the gravitino stays at the `Z/3` identity; and Nieh-Yan returns a `Z`-valued free-part number with zero 3-primary content. The canon already quarantines the underlying homonym -- "geometric torsion (GU contortion) is the antisymmetric connection part, NOT homotopy `Z/n` torsion." **Tag: direction-not-result.** The probe is worth running only as an explicit, on-the-record closure of the pun, with refute the expected outcome.

It also surfaces two concrete, well-posed next computations (the shared-prime-block frame-charge separating test in section 4; the torsion-ON index vs Nieh-Yan probe in section 5), each with a stated confirm/refute. **What it does NOT buy:** it is not a new theorem, and it does not move the verdict off **located, not forced**. The single load-bearing identification underneath it -- RS uniquely straddles the two index types, hence it is the gate -- was already in the computed record; the reframe re-describes it, it does not extend it. The only element with a claim to durable, paper-grade status is the narrow, separately-classified operator fact -- `C = id_14 (x) U` is a frame-fixed internal gauge rotation (frame charge `0.00`), which is the exact computed realization of Weinstein's "gauge rotating one, holding the other fixed because it's tied to the manifold" -- and even that enters only as an illustrative correspondence, not as a result the count depends on.
