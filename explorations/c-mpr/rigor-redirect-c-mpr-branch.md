# Rigor Redirect: C_MPR / Wall / 9-Tuple Branch

Date: 2026-05-31

Purpose: redirect the current C_MPR convergence branch from premature canonization into a theorem-building program. This is a preservation move, not a rejection. The work contains a strong architectural intuition: observer-visible physics may be a monotone or locally certified shadow of a signed, globally constrained, higher-layer provenance substrate. The next pass must separate that intuition into layers with different proof burdens.

## Verdict

The branch should be treated as a candidate architecture and theorem program, not as an already locked category, wall, or formal spine for publication.

Keep:

- The signed-readout / monotone-provenance distinction.
- The idea that local observation can erase global coordination, anomaly, parity, or topology.
- The Hegelian pattern: CALM-like monotone accessibility collides with signed/global physical constraints and synthesizes into a source/readout architecture.
- The usefulness of a 9-tuple or typed checklist as an internal map.

Downgrade:

- "C_MPR is the canonical object" -> "C_MPR is a proposed schema."
- "Same category across cards" -> "convergent schema across cards."
- "Factorization HOLDS rigorously" -> "factorization is trivial unless the accumulator is constrained; useful factorization remains to be proven."
- "CALM is a subcategory" -> "CALM may be a subcase after object and morphism conditions are defined."
- "BvN wall proves the obstruction" -> "BvN is a suggestive obstruction lens that needs categorical hypotheses before it can carry theorem weight."
- "9-tuple locked" -> "9-tuple is a provisional classification frame."

## Layer Discipline

### Layer 1: Provable Core

This is the narrow theorem package that should be made rigorous first.

Define a finite or finitely generated evidence monoid `E`, an ordered target group/vector space `G`, and an additive readout `r: E -> G`. Prove the signed-readout criterion:

- If the readout has only nonnegative coordinates under the chosen evidence order, it is monotone.
- If the readout has both positive and negative components, monotonicity fails under append-only evidence.
- Any signed additive readout decomposes into positive and negative monotone parts through a Jordan/positive-negative factorization.

This theorem is low abstraction and high value because it turns the "signed shadow" intuition into something falsifiable.

### Layer 2: Coordination / Finality Corollary

Once Layer 1 is proven, add a coordination corollary:

- Local monotone access to partial provenance cannot generally decide a signed global predicate until negative or canceling contributions are bounded, finalized, or separately certified.
- This is not yet PCP-blindness. It is a weaker, cleaner result about local append-only observation versus signed global readout.

Proof burden:

- Specify locality.
- Specify finality or bounded missing evidence.
- Specify approximation/error terms if the observable is continuous or probabilistic.

### Layer 3: Concrete Physics Instantiation

Only after Layers 1 and 2 should the branch claim a physics example.

Candidate instantiation:

- Axial charge/chiral anomaly or winding-number accounting in a fixed, cited model.
- Evidence units must be explicit: lattice sites, simplices, modes, plaquettes, paths, or certified local contributions.
- Readout must be explicit: signed sum, index, parity, phase, winding, or obstruction class.
- Observer access must be explicit: what the observer can accumulate locally, what is hidden, and what finalization condition is needed.

The deliverable is not "GU explains anomalies." The deliverable is: "Here is a concrete signed readout where local monotone provenance fails to decide the globally relevant quantity before the right certificate/finality condition."

### Layer 4: C_MPR as Candidate Category

Only after the concrete theorem exists should C_MPR be formalized.

Minimum definition burden:

- Objects: define the tuple fields and their laws.
- Morphisms: define what structure must be preserved.
- Identities: prove the identity morphism satisfies the laws.
- Composition: prove composed morphisms preserve accumulator, readout, certificate, and approximation/error semantics.
- Subcases: only then ask whether CALM-like systems form a full subcategory, reflective subcategory, fibration, indexed category, or none of these.

Until this exists, C_MPR should be cited as a schema, not a category.

### Layer 5: BvN / Adjunction Wall

The Birkhoff-von Neumann lens is useful but currently overburdened.

To turn it into a theorem:

- Define the classical/distributive side.
- Define the quantum/non-distributive or signed/global side.
- Define the functor or adjunction being denied.
- State exactly which preservation requirements are impossible.
- Prove the obstruction without smuggling in the conclusion through object definitions.

BvN by itself says classical distributive event logic does not capture quantum event logic. It does not automatically prove that every signed, stochastic, causal-set, or representation-theoretic readout faces the same wall.

### Layer 6: 9-Tuple / Meta-Template

The tuple is useful as an internal checklist.

It should not be treated as a complete invariant, a theorem, or a publication spine until:

- Each coordinate has an independent definition.
- Coordinate dependencies are marked.
- Examples and counterexamples are classified.
- At least one adversarial card fails gracefully under the tuple.
- A specialist pass checks whether `rr`, `ex`, and `ft` mix algebraic, analytic, and algorithmic layers.

## Premise Corrections

1. "Every observable factors" is not enough.

Any observable can be made to factor if the accumulator is allowed to contain the whole history or the observable itself. The meaningful theorem must constrain the accumulator to be local, finitary, monotone, certifiable, bounded, or computationally limited.

2. "PCP-blindness" is too strong for the current proof.

The current branch has evidence for a local/global certificate tension. PCP-blindness requires a proof model, verifier model, locality condition, soundness/completeness shape, and finality condition.

3. "Anomaly iff" must remain scoped.

Anomaly-free can be sufficient for some monotone local reconstructions, but it is not generally necessary and sufficient across all observables, substrates, or proof models.

4. "Same category" should become "same pattern."

The cards converge on a pattern: source/provenance, observer access, signed/global readout, and certification/finality. That is not yet a category.

5. "CALM" lives at the distributed-computation layer.

Using CALM as a metaphor is productive. Claiming a formal CALM subcategory requires a precise translation between distributed monotonicity and the physical/proof substrate.

## Next Theorem Package

The next rigorous push should produce a small paper-quality core:

1. Signed additive readout theorem.
2. Positive-negative decomposition lemma.
3. Local/final certificate corollary.
4. One concrete physics example.
5. Candidate C_MPR schema with explicitly unproven category obligations.
6. List of conjectures: CALM subcategory, PCP-blindness, BvN adjunction wall, 9-tuple completeness.

## Publication Guardrail

Do not publish C_MPR as the canonical spine yet. Publish, if needed, as:

> a proposed formalization program for observer-accessible monotone shadows of signed/global provenance substrates.

The repo should make the most rigorous path easy for contributors:

- Start with the signed-readout theorem.
- Add examples.
- Mark conjectures clearly.
- Invite counterexamples.
- Separate "proved", "implemented", "suggestive", and "speculative" labels in every card and issue.
