## Section 1. C_MPR — Canonical Definition

**Canonical name.** **C_MPR — the category of Monotone-Provenance + Readout factorization systems.**

### 1.1 Single-paragraph canonical definition

**[speculation, candidate canonical statement; consistent across WRK-388 v3 / WRK-389 v3 / WRK-390 v3]**

> **C_MPR.** An object `M ∈ C_MPR` is a tuple `M = (E, ≤_E, Cert, G, ≤_G, r, P_O)` where:
> - `E` is an evidence / provenance state space, typically a join-semilattice or free commutative monoid generated by local contribution events;
> - `≤_E` is the monotone information order on `E` (append-only refinement);
> - `Cert` is an optional proof-carrying provenance layer (per-site verification predicate, arithmetic-circuit certificate, ZK/HE-inspired proof — populating this layer yields the sub-structure `C_PCP ⊂ C_MPR`);
> - `G` is a semantic codomain (ordered abelian group; integers for index observables, reals for signed charges, more general readout spaces allowed);
> - `≤_G` is the chosen semantic order on `G` with positive cone `G_+`;
> - `r : E → G` is the **final scalar readout**, allowed to be signed, phase-sensitive, or non-monotone;
> - `P_O` is the observer protocol (observation, propagation, merge, proof verification, finality, readout).
>
> A morphism `f : M → M'` is a provenance-preserving map `f_E : E → E'` (monotone with respect to `≤_E` / `≤_E'`) plus a readout-compatibility map `f_G : G → G'` such that `f_G ∘ r = r' ∘ f_E` (exact case) or such that the equation is replaced by an explicitly-bounded approximation relation (ε-approximate / quotient / protocol-mediated case).
>
> **The defining feature of C_MPR** is the *separation* of a monotone evidence-accumulation layer `(E, ≤_E)` from a possibly-signed scalar-readout layer `r : E → G`. The CALM category appears as the **monotone-readout sub-category** `C_CALM ⊂ C_MPR` consisting of objects where `r` is monotone (equivalently: every generator-weight `w(x) ∈ G_+`).

## Section 2. Classical-Value-Lattice Wall Theorem — Canonical Statement

**Canonical name.** **Classical-Value-Lattice Wall Theorem** (a generalization of Birkhoff-von Neumann 1936).

### 2.1 Precise statement

**[speculation, candidate canonical theorem statement; 5-way confirmed across WRK-389 v3 / WRK-388 v3 / WRK-390 v3 / WRK-382 v2 / WRK-384 v2]**

> **(Classical-Value-Lattice Wall Theorem.)** Let `C_ClassicalDistribLR` denote any category of classical local-rule systems whose value space `V` is a *distributive lattice* (covers CALM join-semilattices, signed totally ordered groups like `Z`, PN-counter / G-counter CRDT lattices, generic finite alphabets with classical algebraic structure, Wolfram-CA-class-1-through-4 systems, locally-finite-poset order-ideal lattices arising from Sorkin causal-set substrates, and equilibrium-measure value spaces arising from Parisi-Wu / Hairer / II_1 stochastic substrates). Let `C_GW` denote the category of Ginsparg-Wilson-modified spectral triples with anomaly-free morphisms. Then there is **no 1-categorical adjunction**
>
> ```
> L : C_ClassicalDistribLR ⇄ C_GW : R
> ```
>
> such that `R` preserves non-trivial Dirac-operator content AND `L` lands in the non-degenerate sub-category of `C_GW`.
>
> The **principal obstruction** is the **value-lattice-distributivity vs. projection-lattice-orthomodularity split**. Formally: the projection-lattice functor `Proj : C*-alg → OrthomodLat` factors through `DistribLat ↪ OrthomodLat` only on `Comm-C*-alg ⊂ C*-alg`; the load-bearing `C_GW` objects live outside `Comm-C*-alg`.
>
> The wall is **at the value-lattice level**, not the update-rule level. Loosening monotonicity (signed values, cancellation-bearing observables, signed-BV / PN-counter / Jordan-decomposable readouts) does NOT loosen distributivity. The **only escape** is moving to quantum value spaces (orthomodular projection lattices); this collapses the LHS into the quantum side (`C_QLR`, Heunen-Reyes-style), which is a different paper with a different audience.
>
> The wall is **substrate-agnostic**: it applies wherever the value lattice is classical-distributive, regardless of whether the substrate is a flat hypercubic lattice (WRK-389 v3 source), a Sorkin causal-set (WRK-384 v2 §2), or a stochastic measure space with locally-finite equilibrium structure (WRK-382 v2 §3.2 by composition).

### 2.3 Honest scope

The Classical-Value-Lattice Wall **HOLDS rigorously** as a Birkhoff-von Neumann generalization where the WRK-389 v3 lemma machinery is in place. It **HOLDS by composition / [speculation, application]** where it has been applied to substrates not directly examined by WRK-389 v3 (stochastic in WRK-382 v2; Sorkin in WRK-384 v2). The composition steps are structurally clean but have not been formally written out as a single theorem at the rigor of WRK-389 v3.

- That the computational-irreducibility meta-obstruction (WRK-389 v2) actually applies to GW topology (explicit speculation, do not lean on for theorem statement).
- That all substrate-extensions (e.g., GRW / CSL / Diosi-Penrose collapse models from WRK-382's literature-silent corpus) confront the wall the same way the surveyed substrates do — literature-silent cases remain literature-silent.

## Section 3. Nine-Tuple Meta-Template — Locked

**Canonical form.** The classification framework is the **9-tuple** introduced in WRK-390 v3 as the unified meta-template:

```
M_v3 = (T_X, C_X, U, ρ, C_U, ft, ex, rr, arch)
```

### 3.1 Precise enum definitions for each axis

- **`T_X = (C_X, A_X, ψ_X)`** — shadow no-go theorem (target category, named load-bearing assumptions, conclusion).
- **`C_X`** — shadow category (where `T_X` lives).
- **`U ∈ C_U`** — source object (the candidate evasion / substrate-enriched object).
- **`C_U`** — source category.
- **`ρ : C_U ⇀ C_X`** — readout / shadow operation (partial / forgetful).
- **`ft ∈ {a, b, c, hybrid}`** — failure-mode type:
  - `(a)` non-translation (sub-question 1 fails)
  - `(b)` source falsity (sub-question 2 fails)
  - `(c)` non-reflection (sub-question 3 fails)
  - `(hybrid)` multi-layer failure (≥2 of sub-questions {1, 2, 3} fail simultaneously)
- **`ex ∈ {exact, ε-approx, quotient/collapse, protocol-mediated}`** — exactness mode (v2.1 four-valued replacement for v1's two-valued ε).
- **`rr ∈ {monotone-reducible, signed-bounded-variation, phase-cancellation, computationally-irreducible}`** — readout-reducibility (v2 axis; sub-classifies the readout-side computational regime when `r` is load-bearing).
- **`arch = (P_O, E, r)`** — projection architecture (v2.1 triple): observer protocol, monotone provenance / certificate state, final readout (possibly signed / non-monotone).

### 3.2 Trichotomy + iff for type-(c) ⇔ ε-approx (v3 generalized form)

**Structural lemma.** Type-(c) failure with `ex = exact` is impossible. (If `ρ(U) ∈ C_X` literally and `A_X^i` fails on it, then `T_X`'s antecedent fails on a visible-in-shadow object — the "evasion" is just an object outside the theorem's scope, not a non-reflection.)

**v3 generalized iff (replacing v1's narrow ε-approx-only form):**

> **Type `(ft = c)` ⇔ `ex ≠ exact` AND the failing assumption would be preserved under exact.**

This generalizes v1's "type (c) iff ε-approx + preserved-by-exact" because v3's `ex` is four-valued (`exact / ε-approx / quotient-collapse / protocol-mediated`) rather than two-valued.

### 3.3 Locked — no further changes pending external evidence

The 9-tuple is **locked** as of 2026-05-31 in the sense that:
- All five convergent v3/v2 outputs use it consistently;
- No worker's output adds or modifies an axis;
- The v2.1 broadening (4-valued `ex`, hybrid `ft`) and the v2 sharpening (`rr` axis) are both incorporated;
- The 9-tuple compacts to v1's 7-tuple cleanly when `rr` and `arch` are n/a (single-failure cases at `ft = a` or `ft = b`).

Future revisions should be flagged explicitly; downstream papers may cite the 9-tuple in this locked form.

### 3.4 Verdict on the meta-template itself

**META-FRAME HOLDS — META-THEOREM PARTIAL.** Per WRK-390 v3 §6:
- **As a meta-frame** (rigor checklist + architectural decomposition + structural lemmas about non-reflection), HOLDS.
- **As a meta-theorem** in the strong sense (universal completeness; provable iff between 9-tuple classification and no-go-evasion structure), PARTIAL — universal-completeness conjecture is OPEN.
