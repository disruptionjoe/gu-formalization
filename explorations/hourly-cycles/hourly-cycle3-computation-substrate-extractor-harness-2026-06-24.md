# Hourly Cycle 3 Computation-Substrate Extractor Harness

## 1. Verdict

**Verdict: HARNESS_GATE_ACTIVE_NO_CURRENT_SUBSTRATE_PROMOTION.**

The computation-substrate lane now has an explicit falsification harness. Any
Wolfram/hypergraph, cellular-automaton, stochastic, tensor-network, Type II_1,
or constructor-theoretic proposal must supply a target-free extractor or
constructor certificate before it can claim GU relevance.

No current candidate class satisfies that gate at promotion level. The strongest
current result is negative but useful:

```text
Computation-substrate language is allowed as a search hypothesis.
It is not allowed as a no-go escape or GU derivation without an extractor.
```

Classification decision:

```text
pursue:
  CA/QCA no-go-assumption ledger and complexity/recognition harness work.

protect:
  Wolfram non-bundle substrate insight, tensor/QEC decoder laboratory,
  constructor task specification, GFF/Hairer open gate, and Type II_1 host
  reframing.

park:
  parity-even stochastic and collapse-model baselines until they supply a
  parity/chirality derivation rather than a chosen drift, measure, channel, or
  boundary condition.

kill:
  target-inserted substrate explanations, cardinality-only C3/D4/C_n selectors,
  relabeled Bell/Pauli controls, and parity-odd stochastic terms installed only
  to get the desired output.
```

This does not refute GU or any future computation-substrate route. It blocks a
specific promotion error: replacing a missing geometric proof object with an
undefined computational substrate.

## 2. Harness schema

The harness certificate is:

```text
SubstrateExtractionCertificate-v0
```

It is a finite recognition contract. The certificate may be filled by an
extractor, decoder, coarse-graining map, observer fragment, or constructor task,
but it must name the map and its input data before reading the target output.

Required fields:

| field | required content | automatic failure |
|---|---|---|
| `substrate_data` | Exact rule family, state space, measure/process, tensor/code, Type II_1 object, or task substrate. Must include parameters fixed before target readout. | The substrate is only a metaphor, or its parameters are the target values. |
| `observer_extractor_or_constructor` | Typed map from substrate histories/states/tasks to observer-facing data, with domain, codomain, resource bounds, and provenance. | No map is supplied, or the map is "choose the SM/Bell/chiral output." |
| `locality_causal_update_assumptions` | Locality, causal order, update rule, reversibility/unitarity/Hermiticity, synchrony/asynchrony, boundary, measure, or nonlocality-cost ledger. | A no-go premise is dropped without naming the mathematical or observable cost. |
| `finite_control_output` | Target-free output comparable to finite Connes control: algebra/module/gauge quotient or a declared replacement observer algebra. | `A_F`, `G_SM`, `K_SM`, finite CC tuple, or ordinary SM data are supplied as inputs. |
| `generation_count_output` | Sector count or generation invariant plus replacement tests for `n = 2`, `n = 4`, and preferably `n`. | The proof works unchanged for `C_n`, equal-trace `n` projections, or copied `K_SM tensor C^n`. |
| `chirality_doubling_anomaly_ledger` | Chirality readout, Nielsen-style premise ledger, mirror/doubling verdict, perturbative/global/anomaly checks, and extra-mode accounting. | Chirality is installed through a parity-odd term, boundary, measure, decoder, or SM packet. |
| `CHSH_state_observables_if_claimed` | Source-derived `rho_AB`, Alice/Bob factorization, admissible observables, locality/no-action-channel assumptions, and fixture output. | Bell state, Pauli settings, Pati-Salam labels, or target `CHSH > 2` are imported. |
| `forbidden_target_inputs` | Explicit list of inputs barred from the substrate, extractor, constructor, certificate, and training/selection loop. | The proposal has no forbidden-input screen or uses a forbidden target anyway. |
| `certificate_accepted` | Boolean plus reason. Acceptance requires a target-free extractor/constructor and at least one computed output that survives its replacement controls. | The proposal is host-only, control-only, ansatz-only, target-inserted, or underdefined. |
| `failure_condition` | Exact condition that parks, kills, or rolls back the proposal. | The proposal cannot say what would count as failure. |

Decision procedure:

```text
1. Freeze substrate_data.
2. Screen all inputs against forbidden_target_inputs.
3. Type observer_extractor_or_constructor.
4. State locality_causal_update_assumptions and no-go premise exits.
5. Compute claimed finite-control, generation, chirality/anomaly, and CHSH outputs.
6. Run nearest replacement controls: n = 2, n = 4, parity-paired update,
   Bell/product state controls, and target-free negative controls where relevant.
7. Accept only if the output is computed from the frozen substrate and the
   replacement controls fail by a named obstruction.
```

The certificate can be partial only in the following sense:

```text
accepted_as_gate:
  a CA/QCA or complexity ledger can be promoted as a falsification gate even
  without deriving GU outputs.

accepted_as_GU_relevance:
  requires a target-free extractor or constructor that emits a live GU-facing
  datum, such as finite control, generation count, chirality/anomaly data, or
  a GU-derived CHSH state.
```

## 7. Rollback/falsification conditions

Rollback any future computation-substrate proposal to `underdefined`,
`control_only`, `host_only`, `park`, or `kill` if one of these conditions fires:

1. `observer_extractor_or_constructor` is absent or only described in prose.
2. The substrate, rule search, optimizer, measure, boundary, decoder, or task
   contains a forbidden target input.
3. The claimed generation count survives the `n = 2`, `n = 4`, or `C_n`
   replacement with only labels changed.
4. The finite-control output imports `A_F`, `G_SM`, `K_SM`, the finite CC tuple,
   ordinary SM content, or physical Higgs data.
5. Chirality is installed through a parity-odd drift, chiral boundary, chiral
   Kraus/collapse operator, chosen measure, or target-fitted decoder.
6. A CA/QCA route drops locality, Hermiticity, unitarity, reversibility,
   translation invariance, or on-site chirality without recording the exact
   premise exit and its cost.
7. Extra observer-facing modes appear without a perturbative/global anomaly
   computation.
8. A Bell state, Pauli setting, product/tensor split, or target CHSH value is
   relabeled as GU-derived data.
9. A stochastic, tensor, hypergraph, or constructor route cannot state what
   certificate would terminate the search.
10. A proposal is promoted as GU-relevant while `certificate_accepted` is false.

Positive falsification of this negative verdict would require one accepted
certificate:

```text
certificate_accepted = true
forbidden_target_inputs = passed
observer_extractor_or_constructor = typed and source-derived
at least one live output = computed
replacement_controls = fail by named obstruction
```
