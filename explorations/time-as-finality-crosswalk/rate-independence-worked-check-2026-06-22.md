# Rate-Independence Worked Check: Signed-Readout Monotonicity Criterion

**Status:** Exploration-grade worked check. This document runs the rate-independence test explicitly: for the signed-readout monotonicity criterion, show whether any value of `lambda*(s)` changes the criterion's statement or scope.

**Provenance:** Run 5 (Hostile Rigor Gatekeeper) in `five-run-issuance-rate-observer-contact-2026-06-22.md`, "Heterodox next step" — "Run this test. If `lambda*(s)` is value-free with respect to all GU results, the concept is interesting but not GU-relevant."

**Relation to negative finding:** The negative finding document (`rate-independence-negative-finding-2026-06-22.md`) registers the null result. This document provides the explicit worked check showing HOW and WHY the criterion is rate-independent, so the result is auditable.

---

## 1. The Criterion Under Test

**Signed Readout Monotonicity Criterion** (from `lab/active-research/signed-readout/signed-readout-verdict-2026-05-31.md`):

Let `E` be the free commutative monoid generated by local contribution events `{x_1, x_2, ...}`. The information order on `E` is:

```
e <= e'   iff   e' = e + d   for some d in E
```

(new evidence `d` has been added to `e` to produce `e'`.)

Let `G` be an ordered abelian group with positive cone `G_+`. For each generator `x_i`, assign a weight `w(x_i) in G`.

Define the scalar readout:

```
R : E -> G
R(sum_i n_i x_i) = sum_i n_i w(x_i)
```

**The criterion:** `R` is monotone (order-preserving with respect to the information order on `E` and the group order on `G`) if and only if every generator has nonnegative weight:

```
R is monotone   <=>   w(x_i) in G_+   for every generator x_i
```

## 2. Setup for the Rate-Independence Test

**The issuance rate `lambda*(s)`** is TaF's optimal issuance rate at information state `s`. It is a positive real number — the rate (events per unit time, or events per unit of some other measure) at which new extensions are introduced into the observer's record-bearing network.

In the language of the criterion: `lambda*(s)` is the rate at which new generators `x_i` are added to the generating set of `E`. It is a process-level parameter: it describes the dynamics of the accumulation process, not the algebra or order structure of `E`.

**Question under test:** Does any value of `lambda*(s)` change:

1. The statement of the criterion?
2. The weight function `w` that determines which generators satisfy or violate monotonicity?
3. The information order on `E`?
4. The scope of the criterion (which readouts it applies to)?
5. The failure condition (what would show the criterion cannot support the claimed shadow)?

If the answer is NO to all five questions for every value of `lambda*(s)`, the criterion is rate-independent.

---

## 3. Explicit Check: Does lambda*(s) Appear in the Criterion's Components?

### Check 3.1 — Does lambda*(s) appear in the definition of `E`?

`E` is the free commutative monoid on local contribution events. Its definition requires:
- A generating set `{x_1, x_2, ...}` of local contribution events.
- The free commutative monoid structure (finite sums with non-negative integer coefficients).
- The information order `e <= e'` iff `e' = e + d` for some `d in E`.

None of these involve a rate. The monoid `E` is defined by which events are admissible generators, not by how fast they are added. `lambda*(s)` controls the dynamics of addition to `E`, but the structure of `E` (its generators, order, and algebraic laws) is rate-independent.

**Verdict on 3.1:** `lambda*(s)` does NOT appear in the definition of `E`. Rate-independent. ✓

### Check 3.2 — Does lambda*(s) appear in the weight function `w`?

The weight function `w : {generators} -> G` assigns a group element to each generator. This assignment is determined by the physics of the observable (what contribution each local event makes to the readout). It is not determined by the rate at which events occur.

To make this concrete: in the GW axial-charge example (`lab/active-research/signed-readout/gw-boundary-tests.md`), the weight of a local contribution event `q_A(x)` is the signed real value of the axial-charge density at site `x` in the gauge field `I`. This is determined by the gauge field configuration, not by how fast new gauge configurations are added to the observer's record.

`lambda*(s)` parameterizes the rate of extension introduction. It does not determine the sign, magnitude, or group-element assignment of `w(x_i)` for any generator `x_i`.

**Verdict on 3.2:** `lambda*(s)` does NOT appear in the weight function `w`. Rate-independent. ✓

### Check 3.3 — Does lambda*(s) appear in the information order on `E`?

The information order `e <= e'` iff `e' = e + d` is a partial order on `E` defined purely by the monoid addition. It says: `e'` has more information than `e` if `e'` was obtained from `e` by adding evidence `d`. This is a structural relation, not a dynamical one. The order does not change depending on how fast `d` was added.

`lambda*(s)` determines the rate at which `d` is introduced into the observer's accumulator. But the order relation `e <= e'` holds or fails based on whether `e' = e + d` for some `d`, not based on when or how fast `d` was added.

**Verdict on 3.3:** `lambda*(s)` does NOT appear in the information order. Rate-independent. ✓

### Check 3.4 — Does lambda*(s) change the scope of the criterion?

The criterion applies to all readouts `R : E -> G` of the form "additive readout on a free commutative evidence monoid." Its scope is determined by:
- Whether the evidence accumulation is modeled as a free commutative monoid (required).
- Whether the readout is additive (required).
- Whether an ordered abelian group `G` is specified (required).

These scope conditions are structural, not dynamical. An additive readout on a free commutative evidence monoid has the same scope regardless of how fast new evidence is introduced.

**Edge case to check:** Could a very high issuance rate `lambda*(s)` change whether the accumulation is well-modeled as a free commutative monoid? In principle, if `lambda*(s)` is so high that the observer's accumulator saturates and begins dropping events, the model might break down (the accumulated events would no longer form the full free monoid on the generating set). But this would be a failure of the model's applicability, not a change in the criterion's statement. The criterion would still be correctly stated for the events that ARE accumulated; the scope change would be "criterion applies to fewer events" not "criterion says something different."

Moreover, this saturation scenario belongs to the L2/L4 absorption analysis (observer class and causal order set the saturation threshold), not to a change in the criterion itself.

**Verdict on 3.4:** `lambda*(s)` does NOT change the scope of the criterion. Rate-independent. ✓

### Check 3.5 — Does lambda*(s) appear in the failure condition?

The failure condition for the signed-readout criterion is: some generator `x_i` has negative weight `w(x_i) not in G_+`. This is a condition on the weight assignment, which (per 3.2) does not involve `lambda*(s)`.

The failure mode of the observer-finality protocol (per `explorations/time-as-finality-crosswalk/observer-finality-layer.md`) asks: "What would show the finality protocol cannot support the claimed shadow?" This is different from the signed-readout failure condition, but the signed-readout criterion's own failure condition (negative-weight generator) is rate-independent.

**Verdict on 3.5:** `lambda*(s)` does NOT appear in the failure condition. Rate-independent. ✓

## 6. The Positive Statement of the Negative Result

The rate-independence check is a genuine mathematical result, even though it is negative. It demonstrates:

> The signed-readout boundary theorem (monotone provenance / signed readout split) is a structural result about weight assignments on evidence monoids. It holds equally for observers assembling their records at any rate. Rate-independence is a feature of the theorem's level of abstraction: the theorem concerns the algebra of evidence accumulation, not the dynamics of how that accumulation unfolds over time.

This is analogous to how a topological theorem (e.g., the fundamental group of a space) is independent of the speed at which you traverse the loops — the homotopy type of the loop is what matters, not how fast you run it. The signed-readout criterion concerns the algebraic type of the weight assignment, not how fast generators accumulate.

## 7. Implication for the TaF / GU Interface

The rate-independence result places a precise constraint on where TaF can intersect GU:

**TaF can potentially contact GU only if TaF has structural (not merely dynamical) content.** The issuance rate `lambda*(s)` is a dynamical parameter. GU's active theorem is structural. They do not intersect unless TaF produces a structural result (a bound, an invariant, a class classification) that depends on the issuance rate in a non-trivial way.

The three conditions (A), (B), (C) in the five-run synthesis cross-run verdict are precisely the structural questions: (A) whether `lambda_max` is strictly tighter than L4+L2 structural bounds, (B) whether `lambda_max = Gamma_min` as structural objects in some parameterization, (C) whether the filtered-sheaf temporal obstruction is a new structural object. The rate-independence check shows that none of these conditions is satisfied by the dynamical version of `lambda*(s)` acting on the signed-readout criterion. Only a structural version could make contact.
