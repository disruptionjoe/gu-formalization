# Reciprocal Bridge Ten-Lens Review

## Purpose

Preserve the 2026-06-24 ten-lens review of recent Temporal Issuance, Time as
Finality, and GU formalization findings.

This document is an incorporation note, not a canon update. Its job is to keep
the useful cross-repo findings findable and to state what should, and should
not, be imported into GU.

## Verdict

The reciprocal bridge is useful, but only in bounded roles:

```text
TI/TaF -> GU:
  observer-finality protocol, source/readout discipline, rate-object typing,
  and filtration-sensitive record tests.

GU -> TI/TaF:
  six-axis admission discipline, no-go/absorber style, operator-algebra nulls,
  Ehresmannian source-layer tests, and source-vs-readout failure tensors.
```

It is not currently evidence that Time as Finality explains GU, that Temporal
Issuance supplies GU physics, or that any GU no-go theorem is bypassed.

## What GU Should Keep

### 1. Observer-Finality Sub-Protocol

For any GU candidate that uses observerse, rendering, source/shadow, or smooth
observer-facing shadow language, retain the exploration-only sub-protocol:

```text
source substrate
  -> observer protocol / pairing
  -> record-bearing network
  -> finality relation
  -> signed or semantic readout
  -> smooth 4D observer-facing shadow
```

This does not replace the six-axis protocol. It refines the L2 and L6 fields
when observer-facing language would otherwise remain atmospheric.

### 2. Four-Object Rate Decomposition

The FR series should remain the local authority on rate-language cleanup:

| object | layer | GU use |
|---|---|---|
| `lambda_max` | L2 budget loaded by L4 order-cost | Derived capacity quantity. No promotion. |
| `Delta` | L6 coordination/deadline policy | Candidate finality-sub-protocol field after a second worked example. |
| `Gamma_min` | L1-L2 classicality coupling | Candidate coupling rule only after BvN wall and GU-result sensitivity tests. |
| `O(tau)` | filtered L1/L2 structural obstruction | Best formal bridge target, if a GU readout is filtration-sensitive. |

Important qualification from the relativity/causal-set lens: FR1's
`lambda_max` absorption should be read as conditional on typing the observer
window/access cut `W`, cadence or resource clock, and queue/frontier behavior.
Those fields must not hide a global frontier or preferred observer order.

### 3. Filtered-Readout Bridge

The most promising theorem-shaped target is:

```text
S : Compat_G^MLTT -> FiltSh(C)
R : FiltSh(C) -> ReadoutValues
```

with a witness where `R(S(a))` depends on a transient class
`[omega_tau] in H^1(X, F_tau)` that dies or changes in
`H^1(X, colim_tau F_tau)`.

This would connect:

- TI's formal source witness (`Compat_G^MLTT`);
- TaF's observer/finality and colimit discipline;
- GU's signed-readout or chirality/anomaly readout surface.

Promotion burden: the readout must be GU-relevant and sensitive to the
filtration `{F_tau}`, not only to the final sheaf `F`.

### 4. Classicality Threshold Bridge

`Gamma_min(epsilon) = ln(1/epsilon) / t_obs` and
`lambda_max = 1/t_obs` provide a useful observer classicality bound:

```text
lambda_max <= Gamma / ln(1/epsilon)
```

This is a candidate L1-L2 coupling rule for the six-axis protocol when a
candidate requires an observer to certify a classical shadow. It is not a
replacement for the BvN wall, anomaly analysis, or chirality proof.

### 5. Geometry As Source/Readout Discipline

GU should export, not import, the following method lessons:

- Riemannian vs Ehresmannian: derived connection versus independent connection
  content. Useful for TI's source-layer test.
- Higgs-illusion style projection: useful model of fixed-source disclosure.
- Codazzi / `R_fail`: useful template for separating a coherent observer
  readout from closure of source-level field equations.

These do not solve GU's `R_fail`, Schwarzschild, VZ, Type II_1, or noncompact
Fredholm obligations.

## What GU Should Not Do

- Do not cite TaF or TI as canon for GU.
- Do not treat observer finality as an anomaly, chirality, or spectral-triple
  construction.
- Do not use `lambda_max`, `Delta`, `Gamma_min`, or `O(tau)` to bypass no-go
  theorem assumptions.
- Do not treat a fixed `Y^14 -> X^4` projection as source-side issuance; it is
  closer to the fixed-source projection absorber.
- Do not merge claim ledgers across repos.
