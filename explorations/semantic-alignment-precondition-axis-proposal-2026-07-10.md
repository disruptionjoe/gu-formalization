# Proposed Layer-0 axis: semantic alignment (homonymy check)

Joe's observation: a "no-go" may be like the philosopher's and the geometer's "shape of the object"
-- the same NAME for slightly different meanings across disciplines. The theorem doesn't bite not
because you dropped a structural assumption, but because its key TERM denotes a different object than
the one you built. Proposal: make this a first-class diagnostic run on every no-go.

## Why this is a different KIND of axis than L1-L7 (not "L8")

L1-L7 are all **structural-assumption** axes: which property of the construction is dropped/modified
(substrate, observer, pairing, causal order, emergence, coordination, positivity). The homonymy check
is **semantic/definitional**: it is a relation between the construction's vocabulary and the no-go's
vocabulary, run BEFORE the assumption-decomposition. If the shared terms are homonyms, the entire
L1-L7 mapping is comparing different objects -- apples to oranges. So it is a **Layer-0 precondition**,
not another peer axis. Calling it L8 would miscategorize it as the same kind of thing as L1-L7.

The check, stated: *for each key term in the no-go's statement (the object it constrains, the
predicate/test it uses, the invariant it computes), does the construction's use denote the SAME
mathematical object, or a same-named different one?* It fails in both directions:
- **False escape (equivocation trap):** you believe you evaded, but you quietly weakened the term's
  sense -- the theorem still bites the object you actually have.
- **Real inapplicability (scope by homonymy):** the theorem genuinely constrains a different object;
  your construction is out of its domain, not in contradiction with it.

## This is NOT hypothetical -- it has been the program's load-bearing mechanism

- **Distler-Garibaldi "EVASION BY SCOPE EXIT" (`no-go-class-relative-map.md` 2.4):** GU uses a
  different chirality *invariant* (`ind_H(D_GU)`, operator-analytic) than DG's chirality *test*
  (`V_{2,1}` complexity, Lie-algebraic). Same word "chirality," different object -> the theorem is
  "inapplicable, not contradicted." That verdict IS a Layer-0 homonymy finding, reached case-by-case
  rather than by a named check.
- **The -38 adjudication (`gamma-traceless-38-adjudication-RESULTS.md`):** "the Rarita-Schwinger
  operator" turned out to be TWO published objects -- the physicist's ghost-subtracted gravitino
  (-42) and the geometer's honest gamma-traceless operator (-38). Same name, different object; the
  entire carrier bit is which one "the" operator denotes. A textbook Layer-0 collision.
- **The carrier bit generally:** the whole open question is a name-collision resolved only by SG4.
- **Possibly "generation" itself:** GU's third family is "an imposter for representation-theoretic
  reasons" -- a candidate equivocation between GU's "generation" and the SM's observed generations
  (flagged, not adjudicated).

So the pattern has been operating, unnamed, at the center of the program's biggest results. Naming and
systematizing it is the proposed contribution -- exactly parallel to how E051 systematized the
aggregation-functor point that was happening implicitly.

## Relation to the existing protocol

The map already has a "EVASION BY SCOPE EXIT" *verdict* and an "analogy strength" *column*, but no
systematic, per-term, run-on-every-no-go **semantic-alignment axis**. Scope-exit is a conclusion
reached ad hoc; the homonymy check would be the diagnostic that PRODUCES scope-exit verdicts
predictably and catches the false-escape direction the current protocol has no explicit slot for. It
subsumes and sharpens the scope-exit verdict rather than duplicating it.
