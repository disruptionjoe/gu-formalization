```text

  -> local physical quotient F_phys^b(O)
```

mode images, `rho_AB`, CHSH, and Bell work. It asks whether the repo can state
the exact equivalence/descent/naturality data required without importing the
target QFT structures that the route hopes to recover.

## 4. Equivalence/descent/naturality schema

`SourceDefinedLocalPhysicalFieldEquivalenceRelationAndDescentData_V1` must
contain the following fields.

| field | required data | current repo status |
|---|---|---|
| `branch_context` | branch `b`, local region `O subset X`, restriction policy for `O' subset O`, and observation map or section `iota_b: O -> Y` | partial shell |
| `raw_local_field_object` | typed `R_raw^b(O)` generated from observed pullbacks of `Y`-native GU field content | shell only |
| `source_generators` | explicit generator list for local zero-form and one-form adjoint/spinor data, with support and regularity policy | missing |
| `equations_constraints_policy` | source equations, constraints, redundancy removal, and allowed null/zero-mode identifications | missing |
| `gauge_action_policy` | `H` and inhomogeneous `G` action on `R_raw^b(O)` and compatibility with observation pullback | missing |
| `observer_change_policy` | allowed changes `iota_b -> iota_b'` and induced maps on raw fields | missing |
| `physical_equivalence_relation` | source-defined congruence `~_phys^b(O)` generated by the stated equation, gauge, constraint, null, support, and observer-change relations | missing |
| `physical_quotient` | `F_phys^b(O) = R_raw^b(O) / ~_phys^b(O)` with quotient map `q_O` | not defined |
| `restriction_functoriality` | maps `res_{O,O'}: F_phys^b(O) -> F_phys^b(O')` induced from raw restriction and well-defined on classes | missing |
| `finite_codomain` | source-defined `K_b` with branch, observation, dimension, representation, and bundle/base dependence | representation carrier only |
| `representative_extraction_rule` | representative-level rule `P_raw^b(O): R_raw^b(O) -> K_b` | missing |
| `descent_proof` | proof that `phi ~_phys^b(O) psi` implies `P_raw^b(O)(phi)=P_raw^b(O)(psi)` | missing |
| `descended_extraction` | induced `P_fin^b(O): F_phys^b(O) -> K_b` satisfying `P_fin^b(O) q_O = P_raw^b(O)` | missing |
| `naturality_squares` | commuting squares for restriction, observation pullback, gauge action, observer change, and quotient descent | missing |
| `non_import_guard` | proof that no target Hilbert state, target covariance, Bell/CHSH datum, vacuum, Pauli observable, Gram fit, or ordinary QFT recovery target selects any earlier field | specified as guard only |

Minimal commuting-square obligations:

```text
restriction:
  P_fin^b(O') o res_{O,O'} = k_{O,O'} o P_fin^b(O)

observation pullback:
  P_raw^b(O) o iota_b^* = k_iota o P_Y

gauge action:
  P_raw^b(O)(g . phi) = alpha_g P_raw^b(O)(phi)
  or the invariant special case alpha_g = id

observer change:
  P_raw^{b'}(O)(T_{iota,iota'} phi) = beta_{iota,iota'} P_raw^b(O)(phi)

quotient descent:
  phi ~_phys^b(O) psi => P_raw^b(O)(phi) = P_raw^b(O)(psi)
```

The maps `k_{O,O'}`, `k_iota`, `alpha_g`, and `beta_{iota,iota'}` must be
source-defined. They cannot be chosen to fit later QFT state or Bell data.

pre-extraction object:

derive CHSH, or prove Bell violation. It would only establish the source-side
physical quotient and the rules under which a finite extraction could be tested.
