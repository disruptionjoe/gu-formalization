## 4. First exact obstruction

The first exact obstruction is:

```text
missing_source_operator_action_EL_identity_to_actual_D_GU_epsilon_0_1
```

This is earlier than every VZ computation. Without the source identity from the
manuscript/Oxford material to an actual `D_GU^\epsilon` 0/1
operator/action/EL object, the replay cannot define:

```text
E_actual = P_Q_out sigma_1(D_GU^epsilon) I_Q_in
```

as a source-clean object. The failure is not that `E_actual` has a kernel; the
failure is that `E_actual` is not yet a legitimate object to test for the
actual DGU/VZ family.

The proof-replay block order is:

1. missing source operator/action/EL identity;
2. therefore missing rolled-up domain/codomain;
3. therefore missing principal symbol;
4. therefore missing coefficient/projector/Phi-split checks;
5. therefore no FC-VZ replay or VZ evasion promotion.

## 5. Impact if closed

If a future source certificate closes the first field and then supplies all
downstream fields, the DGU/VZ route becomes a meaningful proof-replay target.
The immediate payoff would be:

- `D_GU^\epsilon` 0/1 would become source-identified rather than typed-spine or
  Oxford-frame inferred;
- the rolled-up 0/1 domain/codomain could be fixed before symbol computation;
- `sigma_1(D_GU^\epsilon)` could be projected to `E_actual`;
- coefficients `a`, `b`, and nonzero `lambda_d` could be checked instead of
  assumed;
- `Phi_d` and `Phi_F` could be separated by order;
- Q-sector projectors/import maps could be normalized;
- FC-VZ-1 style kernel checks and later FC-VZ-4 subprincipal/constraint checks
  could be attempted without target-import contamination.

This would not by itself prove full VZ evasion. It would only open the first
legitimate actual-operator replay.
