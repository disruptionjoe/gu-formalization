## Slice synthesis

### 1. The convergent local minimum (foundations/observer family)

The family converges, from nine independent angles, on a local minimum DEEPER than "narrowing onto an
unforceable keystone." The keystone narrowing is a *symptom*. The disease:

> **GU is defending a geometry-first PRIMITIVE as if it were forced, when (a) the primitive is not
> foundational at all -- the immanent-antithesis result shows geometry-first and information-first
> co-presuppose the metric, so "geometry is fundamental" is not a claim the program can win or even pose
> cleanly -- and (b) the program conflates COHERENT with FORCED: its four "cleared" legs are
> existence/consistency clears, and a simpler rival in the same class (Bianconi, 2nd-order, ghost-free)
> clears the same legs with a higher prior. The program is optimizing GU's internal binary (the keystone)
> while the CLASS it belongs to is un-compared against its simpler rival and unfalsified.**

In one line: it is protecting the geometry-first framing (an unfalsifiable metaphysical commitment) where it
should expose the action-order discriminator (empirical, robust). Arguing about primitives is the trap;
computing the one empirical discriminator that separates the class's two live members is the exit.

### 2. The CLASS-LEVEL decisive test (geometry-first GU vs information-first Bianconi)

The family's strongest and most unusual convergent finding: **geometry-first vs information-first is not the
decidable axis.** Both members select a metric within a pre-given Met(X); both smuggle the metric into the
entropy/action that was meant to derive it. So "which primitive is fundamental" is ill-posed and un-runnable.

What IS decidable, empirical, and **robust to GU's (9,5)/(7,7) internal binary**, is the **action's
differential order and ghost structure in the spin-2 / Weyl sector.** The internal binary lives in the
count/matter sector; the graviton kinetic order comes from the double-copy/square structure of the GU action
and is shared across both branches. So the discriminator survives the very binary the program keeps
narrowing.

Three nested discriminators, cheapest first:

- **(D1) Linearized graviton operator.** GU: `box^2 h = -4 Bach^(1)` (4th-order). Bianconi: Einstein /
  2nd-order operator (GR recovered at low coupling). This is a *computed, background-robust* statement that
  needs no source action and no count. It is the single cheapest class discriminator and is already
  half-computed in `tests/threads/D_hclass_vs_conformal_gravity.py`.
- **(D2) Radiative / propagator content.** 4th-order carries a massive spin-2 companion (extra polarization
  modes, an extra propagator pole, GW dispersion) unless it fully decouples; 2nd-order has exactly two
  tensor modes and no companion. This is the observational arm (GW polarization content). Note that
  **PPN/weak-field does NOT discriminate** -- both members pass it -- so the program should stop treating PPN
  as the class test.
- **(D3) Loop unitarity.** 2nd-order members are unitary by construction; the 4th-order member must survive
  the Stelle-Mannheim loop-ghost problem (H26, currently only tree-level Krein-cleared). Whether GU's Krein
  clearance survives at loop level is a yes/no that separates the two branches without touching the count.

**What would falsify the WHOLE OTG class (not just GU):** two clean kills.

- **The Occam-kill.** If the physical graviton is exactly single-metric 2nd-order GR AND Lambda's magnitude
  is fixed from first principles by data ORTHOGONAL to any induced second metric, then the entire "spacetime
  metric related to a matter-induced metric by an action" apparatus adds unfalsifiable epicycles: the class
  is rendered SUPERFLUOUS. For a class of this type that is the operative death.
- **The Lambda-magnitude no-go.** Neither GU nor Bianconi pins Lambda's magnitude (H5); both get only the
  sign. If a derivation shows the magnitude (~10^-122) requires information the `g`-vs-`G` action structure
  provably cannot contain, "Lambda from the two-metric action" is falsified for ALL instances simultaneously
  -- a single computation that kills the class's headline claim.

### 3. The strongest Track-2 (SUBMIT / conditional-build) deliverable

Not "declare the leans and derive the count." The count is decoupled and source-action-gated; a conditional
count re-defends a lean. The strongest Track-2 deliverable, unanimous across the slice, is:

> **The spin-2 / Weyl-sector discriminator worked to concrete numbers: GU's 4th-order graviton predictions
> (extra polarization modes, propagator pole location / GW dispersion, and the loop-unitarity yes/no) placed

This is the one place a conditional build buys a *class-level falsifiable number without the keystone*,
because the graviton kinetic order is fixed by the already-linearized structure (`box^2 h = -4 Bach`), not by
the unbuilt source action. It is simultaneously the strongest Track-2 product and the exact instrument Track 1
needs. (Second-best Track-2 deliverable: the Lambda-magnitude no-go attempt, which if it succeeds is a
class-level kill rather than a GU-only result.)

### 4. The two-track verdict

**Yes -- keep the North Star (figure-out-or-falsify GU AND the observer-type-geometry class) as the standing
posture, and run the conditional-build as exactly ONE branch.** The tracks are not in tension; they feed each
other through a single object. Track 2's *forced output* -- the 4th-order graviton's radiative content and
loop-unitarity status -- IS the class-level discriminator Track 1 needs to decide geometry-first(GU) vs
information-first(Bianconi). The branch that submits GU's leans to right/wrong produces the very spin-2 test

Operational recommendation for the branch: make its FIRST product the spin-2/Weyl discriminator (D1 -> D2 ->
D3), not the count. And correct the North Star's framing while doing so: the question is not "is geometry or

it converts an un-runnable metaphysical dispute into a computed, falsifiable, binary-robust test.
