# Construction-Space Pre-Registered Endings

This note freezes how to read the two possible endings of
`CONSTRUCTION-SPACE-EXPLORATION` before the map accumulates more sector-fit
evidence. It changes search interpretation only. It does not move claim status,
canon verdicts, proof status, public posture, paper status, or external action.

## If The Joint R0 Intersection Empties

The permitted statement is:

> The predeclared construction space, tested against the frozen sharp constraint
> lists at Rung 0 consistency strength, has empty joint intersection.

That is a falsification-shaped result for the predeclared construction class,
not a global verdict on GU, observerse geometry, or any construction outside
the map. It must name:

- the map revision and track-cell denominator;
- which cells failed which sector constraint lists;
- which cells were `GATED` or `UNDERDEFINED`, if any;
- which constraint lists changed after the initial freeze, if any;
- whether the council can name a reparameterization that explains more rather
  than merely surviving.

An empty intersection is handed to daily stewardship and Joe before any
exhaustion verdict, public framing change, or canon movement. The repo may still
bank the bounded result as construction-class evidence.

## If A Survivor Emerges

The permitted statement is:

> At least one mapped construction family survives the tested sector tracks at
> the recorded minimum rung, with the recorded typed imports.

Claim strength equals the weakest rung across the sectors tested:

- `R0`: not falsified by the sharp constraint lists. This is consistency only.
- `R1`: hosted with typed and counted imports. This is existence-with-imports,
  not native unification.
- `R2`: native selection with no sector-specific repair. This is the North Star
  bar, and only this rung supports the strong unification reading.

A survivor at joint `R1` or better on two or more sectors becomes a frozen
construction candidate and enters the recovery-certification ladder. It does
not erase the original no-go that carved the map.

## Import Counter

Use these types consistently in map notes, candidate receipts, and council
handoffs:

| Type | Counts In N? | Meaning |
|---|---:|---|
| `native` | 0 | Emitted by the construction before target comparison. |
| `forced` | 0 | Follows from named native premises without adjustable choice. |
| `assumed` | 1 | Granted as an explicit branch or adapter axiom. |
| `imported` | 1 | Supplied from outside the construction but not fitted to the target data. |
| `fitted` | 1 plus flag | Chosen using target data; cannot support prediction or native selection. |
| `comparator` | invalid as evidence | Standard physics object used only as a baseline or vocabulary. |
| `gated` | unresolved | Blocked on a named dependency; count only after resolution. |
| `underdefined` | unresolved | Not typed enough to support rescue, no-go, or import counting. |

Counting rules:

- Count imports per sector and per cell.
- Reuse one import across sectors only when the same typed object and provenance
  do the work in each sector.
- A sector-specific adjustable identification counts separately in every sector
  it repairs.
- Target-shaped stress, spectrum, or parameter choices do not count as R0
  consistency evidence; they fail the shortcut ban.
- A comparator may sharpen a constraint list, but it never becomes GU evidence.

## Guardrail

This addendum pre-registers interpretation, not physics. It is a guard against
post-hoc strengthening in both directions: an empty map is not a global GU kill,
and a surviving map cell is not a unification claim unless it earns Rung 2.
